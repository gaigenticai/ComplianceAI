🔍 Regulatory Intelligence Agent API

Comprehensive API Documentation for Regulatory Feed Monitoring and Processing

Version: 4.0 | Base URL: http://localhost:8002 | Last Updated:

📋API Overview

🔍 Regulatory Intelligence Agent: This agent continuously monitors regulatory feeds from authorities like EBA, ESMA, BaFin, and CBI, parsing documents to extract compliance obligations and regulatory updates.

🎯 Core Capabilities

📡 Feed Monitoring

  • Real-time regulatory feed monitoring
  • Multi-source integration (EBA, ESMA, BaFin, CBI)
  • Automated feed scheduling and management
  • Resilience and error handling
  • Feed health monitoring and alerts

📄 Document Processing

  • AI-powered document parsing
  • Multi-format support (PDF, HTML, XML)
  • Natural language processing
  • Entity recognition and extraction
  • Content classification and tagging

⚖️ Obligation Extraction

  • Automated obligation identification
  • Regulatory requirement extraction
  • Deadline and timeline detection
  • Jurisdiction-specific processing
  • Compliance impact assessment

🔄 Integration

  • Kafka message bus integration
  • RESTful API endpoints
  • Webhook notifications
  • Database persistence
  • Real-time status updates

# API Key Authentication curl -H "Authorization: Bearer YOUR_API_KEY" \ -H "Content-Type: application/json" \ http://localhost:8002/api/v1/feeds # OAuth 2.0 Authentication (Production) curl -H "Authorization: Bearer YOUR_OAUTH_TOKEN" \ -H "Content-Type: application/json" \ http://localhost:8002/api/v1/feeds

📡Feed Management API

GET /api/v1/feeds
Retrieve all configured regulatory feeds with their current status and monitoring information.

Query Parameters

Parameter Type Required Description
status string optional Filter by feed status: active, inactive, error
jurisdiction string optional Filter by jurisdiction: EU, DE, IE, FR, etc.
authority string optional Filter by regulatory authority: EBA, ESMA, BaFin, CBI
limit integer optional Maximum number of feeds to return (default: 50)

POST /api/v1/feeds
Create a new regulatory feed for monitoring. The system will automatically start monitoring the feed based on the specified frequency.

Request Body

Field Type Required Description
name string required Human-readable name for the feed
authority string required Regulatory authority (EBA, ESMA, BaFin, CBI, etc.)
jurisdiction string required Jurisdiction code (EU, DE, IE, FR, etc.)
url string required RSS/Atom feed URL or API endpoint
update_frequency string optional Update frequency (15m, 30m, 1h, 6h, 24h) - default: 1h
feed_type string optional Feed type: rss, atom, api, scraper - default: rss
authentication object optional Authentication configuration if required
curl -X POST http://localhost:8002/api/v1/feeds \ -H "Authorization: Bearer YOUR_API_KEY" \ -H "Content-Type: application/json" \ -d '{ "name": "ECB Banking Supervision Updates", "authority": "ECB", "jurisdiction": "EU", "url": "https://www.bankingsupervision.europa.eu/rss", "update_frequency": "30m", "feed_type": "rss", "authentication": null }'
PUT /api/v1/feeds/{feed_id}
Update an existing regulatory feed configuration. Changes take effect on the next scheduled check.

Path Parameters

Parameter Type Required Description
feed_id string required Unique identifier of the feed to update
DELETE /api/v1/feeds/{feed_id}
Delete a regulatory feed. This will stop monitoring and remove all associated data.
⚠️ Warning: Deleting a feed will permanently remove all historical data and extracted obligations. This action cannot be undone.
POST /api/v1/feeds/{feed_id}/trigger
Manually trigger an immediate update of a specific feed, bypassing the scheduled frequency.

Response Example

{ "message": "Feed update triggered successfully", "feed_id": "eba_banking_updates", "trigger_time": "2024-01-15T10:45:00Z", "estimated_completion": "2024-01-15T10:47:00Z", "status": "processing" }

📄Document Parsing API

POST /api/v1/documents/parse
Parse a regulatory document to extract obligations, requirements, and metadata using AI-powered natural language processing.

Request Body

Field Type Required Description
document_url string optional URL of the document to parse
document_content string optional Base64 encoded document content
document_type string required Document type: pdf, html, xml, docx
authority string required Source regulatory authority
jurisdiction string required Applicable jurisdiction
language string optional Document language (auto-detected if not provided)
extract_obligations boolean optional Whether to extract compliance obligations (default: true)
curl -X POST http://localhost:8002/api/v1/documents/parse \ -H "Authorization: Bearer YOUR_API_KEY" \ -H "Content-Type: application/json" \ -d '{ "document_url": "https://www.eba.europa.eu/sites/default/documents/files/document_library/Publications/Guidelines/2024/EBA-GL-2024-01.pdf", "document_type": "pdf", "authority": "EBA", "jurisdiction": "EU", "extract_obligations": true }'

Retrieve regulatory obligations with advanced filtering and search capabilities.

Query Parameters

🔗Webhooks & Notifications

POST /api/v1/webhooks
Create a webhook to receive real-time notifications about regulatory updates and processing events.

Request Body

Field Type Required Description
url string required Webhook endpoint URL
events array required Events to subscribe to
secret string optional Secret for webhook signature verification
active boolean optional Whether webhook is active (default: true)

Available Events

  • feed.updated - New content detected in a regulatory feed
  • document.parsed - Document parsing completed
  • obligation.extracted - New regulatory obligation identified
  • feed.error - Error occurred while processing a feed
  • system.health - System health status changes
curl -X POST http://localhost:8002/api/v1/webhooks \ -H "Authorization: Bearer YOUR_API_KEY" \ -H "Content-Type: application/json" \ -d '{ "url": "https://your-app.com/webhooks/regulatory", "events": ["obligation.extracted", "feed.error"], "secret": "your-webhook-secret", "active": true }'

Webhook Payload Example

{ "event": "obligation.extracted", "timestamp": "2024-01-15T10:45:00Z", "webhook_id": "wh_12345", "data": { "obligation_id": "eba_gl_2024_01_001", "title": "ICAAP Documentation Requirements", "authority": "EBA", "jurisdiction": "EU", "priority": "HIGH", "effective_date": "2024-07-01", "source_document": { "document_id": "doc_eba_gl_2024_01", "title": "Guidelines on Internal Capital Adequacy Assessment Process", "url": "https://www.eba.europa.eu/sites/default/documents/files/document_library/Publications/Guidelines/2024/EBA-GL-2024-01.pdf" } } }
🔐 Webhook Security: All webhook payloads are signed using HMAC-SHA256. Verify the signature using the X-Signature header to ensure authenticity.
