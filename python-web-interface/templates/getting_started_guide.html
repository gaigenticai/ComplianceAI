Getting Started Guide

Welcome to ComplianceAI! This guide will help you get started with the platform quickly and efficiently.

1. Prerequisites

System Requirements

Component Minimum Recommended
CPU 4 cores 8+ cores
RAM 8 GB 16+ GB
Storage 50 GB SSD 100+ GB SSD

Software Dependencies

  • Docker Compose: Version 2.0 or later

2. Installation

Step 1: Clone the Repository

git clone https://github.com/your-org/complianceai.git
cd complianceai

Step 2: Environment Configuration

# Copy the example environment file
cp .env.example .env

# Edit the environment file with your settings
nano .env

Essential Configuration

# Basic Settings
ENVIRONMENT=development
REQUIRE_AUTH=false

# Database Settings (default values are usually fine for development)
POSTGRES_HOST=postgres
POSTGRES_PORT=5432
POSTGRES_DB=kyc_db
POSTGRES_USER=postgres
POSTGRES_PASSWORD=password

# Redis Settings
REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=password

# OpenAI API Key (REQUIRED for document processing)
OPENAI_API_KEY=your-openai-api-key-here

# Kafka Settings (usually default values work)
KAFKA_BOOTSTRAP_SERVERS=kafka:9092

Step 3: Start the System

# Start all services
docker-compose up -d

# Wait for services to be ready (may take 2-3 minutes)
docker-compose logs -f --tail=50

Step 4: Verify Installation

# Check service health
curl http://localhost:8001/health

# Check service status
docker-compose ps

# View system logs
docker-compose logs --tail=20

Step 5: Access the Web Interface

Open your web browser and navigate to: http://localhost:8001

4. Your First KYC Case

Step 1: Access the Dashboard

Navigate to the main dashboard at http://localhost:8001

Step 2: Create a Customer Profile

  1. Click on "New Customer" or "Add Customer"
  2. Fill in basic customer information:
    • Customer ID (unique identifier)
    • Customer Type (Individual or Corporate)
    • Basic contact information
  3. Save the customer profile

Step 3: Submit KYC Documents

  1. Select the customer from the customer list
  2. Click "Start KYC Process" or "Upload Documents"
  3. Upload required documents:
    • Government-issued ID (passport, driver's license)
    • Proof of address (utility bill, bank statement)
    • Financial documents (if applicable)
  4. Specify the regulatory jurisdiction (EU, US, etc.)
  5. Submit for processing

Step 4: Monitor Processing

  1. View the processing status on the dashboard
  2. Check the progress indicators for each processing stage
  3. Review any alerts or notifications
  4. Wait for automated processing to complete

Step 5: Review Results

  1. Once processing is complete, review the KYC decision
  2. Check the compliance assessment
  3. Review any recommendations or required actions
  4. Approve or escalate the case as needed

9. Common Workflows

Standard KYC Process

  1. Customer Registration: Create customer profile in the system
  2. Document Collection: Upload required identification documents
  3. Automated Processing: System performs OCR, validation, and risk assessment
  4. Compliance Check: Apply regulatory rules and requirements
  5. Decision Making: System or human makes final approval decision
  6. Reporting: Generate required regulatory reports
  7. Customer Notification: Inform customer of the outcome

High-Risk Case Handling

  1. Risk Identification: System flags case as high-risk
  2. Enhanced Due Diligence: Additional verification steps required
  3. Manual Review: Compliance officer reviews the case
  4. Additional Documentation: Request additional supporting documents
  5. Escalation Process: Escalate to senior compliance officer if needed
  6. Final Decision: Make approval/rejection decision with detailed reasoning

Regulatory Reporting

  1. Report Generation: System automatically generates required reports
  2. Data Validation: Ensure all required data is included and accurate
  3. Format Compliance: Verify reports meet regulatory format requirements
  4. Quality Assurance: Review reports for completeness and accuracy
  5. Submission: Submit reports to regulatory authorities
  6. Confirmation: Receive and log submission confirmations

Webhook Integration

Setting Up Webhooks

# Register webhook for case completion notifications
webhook_config = {
    'url': 'https://your-app.com/webhooks/kyc-complete',

response = requests.post('http://localhost:8001/api/webhooks',
    json=webhook_config)

Webhook Handler Example

from flask import Flask, request, jsonify
import hmac
import hashlib

app = Flask(__name__)
WEBHOOK_SECRET = 'your-webhook-secret'

@app.route('/webhooks/kyc-complete', methods=['POST'])
def handle_kyc_complete():
    # Verify webhook signature
    signature = request.headers.get('X-Webhook-Signature')
    payload = request.get_data()

    expected_signature = hmac.new(
        WEBHOOK_SECRET.encode(),
        payload,
        hashlib.sha256
    ).hexdigest()

    if not hmac.compare_digest(signature, expected_signature):
        return jsonify({'error': 'Invalid signature'}), 401

    # Process the webhook payload
    data = request.get_json()
